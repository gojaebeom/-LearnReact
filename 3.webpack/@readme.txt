웹팩의 기본 개념 이해하기

- 파일 내용을 기반으로 파일 이름에 해시값을 추가합니다. -> 효율적으로 브라우저 캐싱을 이용합니다.
- 사용되지 않는 코드를 제거합니다.
- 자바스크립트파일을 압축시킵니다.
- JS에서 CSS. JSON, 텍스트 파일 등을 일반 모듈처럼 불러올 수 있습니다.
- 환경 변수를 주입합니다.

웹팽을 사용하는 가장 큰 이유는 모듈 시스템(ESM, commonJS)을 사용하기 위해서 입니다.

----------

예전의 웹사이트는 동적인 처리가 별로 없기 때문에 html 파일에서 script 태그를 나열하여 
자바스크립트 파일을 관리하였습니다. 하지만 점점 웹이 점점 동적인 처리가 많아지면서 
자바스크립트 파일을 관리가 어려워 졌습니다. 이 와같은 단점들은 모듈 시스템을 사용하면서 해결이 가능해졌습니다.

----------

요즘 브라우저는 ESM을 지원합니다. 하지만 오래된 브라우저와 많은 오픈 소스가 commonJS로 작성되어있습니다.
(commonJS는 요즘 브라우저가 인식하지 못합니다.)

----------

이전까지 react 파일을 다운받아 사용하였지만 웹팩을 사용하게 되면서(모듈시스템을 사용할 수 있게되면서) npm을 이용한 
관리를 할 수 있게 되었습니다.
